13.4 - Changes in tool interface files (.mpd)

  • [<CR636155> and <CR627138>] MPD limited vector width of parameter C_DQS_LOC_COLx to 64 bits. If C_NDQS_COL_LOCx > 8 for any configuration, then the MSB values specified in C_DQS_LOC_COLx exceeding 64 bits and be cut off resulting in synthesis errors.

  • [<CR627533>] Typo fixed on default value of C_RD_WR_ARB_ALGOIRTHM. The default value changed to "RD_PRI_REG" from "RD_PRI_PREG". Fixes potential system performance issues in certain configurations that may have been exhibited in the axi_v6_ddrx_v1_04_a pcore only.

13.3 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn)

 
New Features:

  • Updated V6 MIG PHYs and GUI to MIG release version 3.9.

  • See MIG version 3.9 release notes and documentation for more information about changes to MIG, supported FPGA devices, supported memory components, and board design information.

  • [<CR618420>] Default value of C_BYPASS_INIT_CAL changed from "OFF" to "FAST". This allows the user to have this set automatically for simulation. During synthesis this parameter will be set to "OFF" in mem_intfc.

Note: Unless specified, limitations and resolved issues affect all previous versions.

12.4 - Changes in VHDL/Verilog/Netlist sources (.vhd, .v, .ngc, .edn)

 
New Features:

  • AXI Read/Write arbitration algorithm modified to improve throughput during simultaneous AXI read and write.

  • Incremental size reduction and timing improvement.

Resolved issues:

  • [<CR573874>] Fix issue where EDK calculated memory clock frequency is not passed to the MIG GUI.

  • [<CR574199>] IODELAY_GRP information passed to IDELAYCTRL elements now reflects axi_v6_ddrx instance name.

Known Issues / Limitations:

  • Multiple AXI V6 DDRx instances cannot be instantiated in the same EDK design.

12.4 - Changes in tool interface files (.mpd)

  • Core state updated from Pre-Production to Production status.
